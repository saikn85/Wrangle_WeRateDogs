Introduction

This project talks about Data Wrangling that involves three distinct steps that can be revisited anytime; i.e. it is iterative if your needs are not met. The three steps are:

  1. Gathering
  2. Assessing
  3. Cleaning

Personally this project was the longest that I had ever worked on, it took me about 10 days to finish this. The delay was because of the application for security tokens for Twitter API.


Challenges

The following were the issues that I had to over come.

Gathering


  • Querying the API was bit troublesome as on an average it took about 30 minutes to fetch data.
  • Querying the API for missing records: I thought that there was something wrong with my code, so I redid the entire thing, only to find out that the records had been deleted and there was nothing wrong with the code.

Assessing


  • This was easy, it took about an hour or so find faults and criticise it for all the mess it was in.
  • Although, assessing the image-predictions.tsv file was a bit overwhelming as there was nothing wrong in it, until programmatic assessment was done.

Cleaning


  • Well, this was the longest of all the processes. Defining the cleaning goals initially was tough, but it all got easy after two attempts.
  • Another challenge that I had to encounter was defining the regular expressions for extracting name from text, and to find those, that were missed by regexs'.

Overall Experience

Well, this one will teach you to learn the importance of cleaning, gathering and assessing data before we start building model or performing EDA.

It is strongly recommended that every individual must take a refresher course in python and pandas, after all you get used to R's syntax and it ability to handle untidy data.
